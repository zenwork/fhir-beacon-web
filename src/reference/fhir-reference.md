# FHIR Specification Coverage

The FHIR Beacon library is designed to closely map the FHIR specification. It 
aligns with the published FHIR.

* [data types](https://hl7.org/fhir/datatypes.html) 
* [resource list](https://hl7.org/fhir/resourcelist.html).

FHIR Beacon components map semantically to the official FHIR surface. Status
labels describe the current fhir-beacon implementation for that FHIR concept;
they do not define the scope of the reference list.

FHIR version: `5.0.0`

## Status Labels

- <wa-badge variant="success" pill>available</wa-badge>: a dedicated renderer or
  semantic base renderer exists and is usable for the named FHIR surface.
- <wa-badge variant="brand" pill>preview</wa-badge>: a public renderer exists
  and has Storybook coverage, but the implementation is still maturing.
- <wa-badge variant="warning" pill>partial</wa-badge>: generated model or data
  type support exists, but there is no dedicated public renderer yet.
- <wa-badge variant="neutral" pill>planned</wa-badge>: the FHIR specification
  includes the item, but there is no dedicated renderer yet.
- <wa-badge variant="danger" pill>internal</wa-badge>: used internally by the
  library, but not currently part of the public API.
