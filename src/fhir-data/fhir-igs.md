# FHIR IGs

FHIR IGs covers implementation-guide-driven FHIR: profiles, extensions,
terminology, and constraints.

This chapter is for users asking:

- My data conforms to an IG profile. Can the UI reflect that?
- How are extensions displayed or customized?
- How does FHIR Beacon handle terminology and coded values?
- Can constraints or profile-specific rules affect rendering?

## Current Coverage

| Area        | Status                                                 | Notes                                                       |
| ----------- | ------------------------------------------------------ | ----------------------------------------------------------- |
| Profiles    | <wa-badge variant="brand" pill>experimental</wa-badge> | Profiling support exists and is still evolving.             |
| Extensions  | <wa-badge variant="brand" pill>experimental</wa-badge> | Extension rendering exists, including profile-related work. |
| Terminology | <wa-badge variant="brand" pill>experimental</wa-badge> | Valuesets and codesystems support exists.                   |
| Constraints | <wa-badge variant="brand" pill>experimental</wa-badge> | Profile validation support exists and is still changing.    |

FHIR IG support is an important part of the v1 direction. The preview
documentation introduces the areas that exist today and will expand as the
implementation stabilizes.
